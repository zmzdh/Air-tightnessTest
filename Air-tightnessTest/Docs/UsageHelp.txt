# 腰托按摩控制板检测系统使用帮助

本帮助文档基于当前版本的程序实现编写，旨在帮助操作员、工程师与管理员正确使用腰托按摩控制板检测系统的各项功能。

## 登录与权限管理
- 系统启动时会弹出登录窗口，未登录的用户无法进入主界面。登录成功后，标题栏会显示“当前用户”信息。
- 系统按照角色划分权限：
  - **Admin**：拥有全部菜单权限，可访问测试、手动调试、机型配方、生产报表、用户管理、系统设置、系统日志以及使用帮助。
  - **Engineer**：可访问测试、手动调试、机型配方与生产报表菜单，其它管理项会被隐藏。
  - **Operator**：仅可访问测试界面与生产报表，适用于日常产线操作。
- 无论角色如何，“系统日志”与“使用帮助”入口始终可见，用于快速排障与查看说明。

## 导航菜单与状态栏
- 顶部导航栏提供常用功能入口，点击按钮即可在主内容区域加载对应用户控件。
- 底部状态栏实时显示 PLC 连接状态、MES 连接状态与系统时间，方便追踪设备通信情况。

## 测试界面
- 进入“测试界面”后，系统会加载产品型号列表（`ConfigService.LoadProductModelsAsync`）并记录上一次选择的型号与工单号。
- 请先确认并填写以下信息：
  1. **产品型号**：在下拉框中选择需要测试的机型；系统会自动展示配方图片与参数。
  2. **工单号**：默认为上次保存的工单，可手动修改；控件会在测试结束时调用 `SavePersistentStateAsync` 保存记录。
  3. **条码**：若所选型号启用了条码校验，则必须先扫码，否则 `StartChannelAsync` 会提示未扫描条码。
- 测试支持左右两个通道独立运行：
  - 点击“左侧启动”或“右侧启动”按钮后，控件会校验参数、刷新最新配方，并调用 `TestService.StartTestAsync` 启动测试。
  - `ChkContinueDuplicate` 可控制是否允许重复条码继续测试。
  - 测试过程中，界面会根据 `TestService` 回调实时更新阶段列表、状态横幅与 PLC 反馈。
- 测试完成后，主窗口会在 `TestService_OnTestCompletedSaveRecord` 中自动保存记录到数据库，并尝试推送 MES，如发送失败会在状态栏给出提示。

## 手动调试
- “手动调试”界面通过 `CommService` 使用串口设备1直接发送 485 报文，实时更新接收区数据。
- “发送数据”区域仅保留一个发送框，点击“单次发送”即发送一帧报文；点击“连续发送”后每 200ms 发送一次，点击“停止连发”终止发送。
- 接收区按报文地址字节映射到通道 1-4，地址为 0 或非法时不显示。
- 所有手动按钮均会记录当前状态，界面也会从主窗口获取最新的 PLC 数据并刷新显示，避免与自动测试冲突。

## 机型配方
- “机型配方”用于维护产品型号、按摩流程与报文配置：
  - 支持为每个型号配置上升/下降、同步充放气等腰托动作序列（`LumbarActionEntry`）。
  - 可编辑按摩测试参数（`MassageTestSettings`），包含总时长、电流峰值/均值等阈值。
  - 通过报文配置对话框（`MessageConfigDialog`）维护上下电、睡眠、按摩等指令，并可同步保存到数据库供测试时加载。
- 修改完成后请保存，以便测试界面获取最新配方；系统也支持从数据库重新加载现有配置。

## 生产报表
- 报表界面提供按时间、机型、操作员、工单与条码的多条件查询，默认展示最近 7 天数据。
- 查询结果包含统计概览（总数、合格数、不良数、合格率、平均测试时长）以及分页列表，方便快速定位记录。
- 点击“导出”按钮可生成 CSV 或 Excel（通过先导出 CSV）的数据文件，便于长期归档或进一步分析。

## 用户管理（管理员）
- “用户管理”菜单仅对管理员开放。控件会再次校验当前登录用户角色，确保只有 Admin 能继续编辑。
- 支持查看用户列表、启停账号、重置密码，并维护角色、部门与功能权限，同时展示登录次数与测试次数统计。

## 系统设置（管理员）
- 系统设置面板用于维护 PLC 与 MES 通信参数：
  - 支持读取与保存 PLC IP、端口及 MES 地址、协议等信息（`ConfigService.SaveAppConfigAsync`）。
  - 可直接在界面发起 MES 连接测试，连接成功后状态标签会转为绿色。
  - 若修改了 PLC 参数且当前处于连接状态，系统会提示重新连接以应用新配置。

## 系统日志
- 系统日志实时订阅 `LogService` 的日志事件，默认展示最新 1000 条信息，并支持 Info、Warning、Error 级别筛选。
- 可使用“刷新”按钮重新加载历史记录，“清空”按钮会调用 `LogService.ClearLogs()` 清除缓存，便于排查设备异常。

## 使用帮助
- “使用帮助”入口面向所有登录用户开放。控件会读取 `Docs/UsageHelp.txt` 并展示在主界面。
- 如果文档内容有更新，可点击右上角的“重新加载”按钮即时刷新显示。
